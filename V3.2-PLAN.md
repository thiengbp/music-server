# v3.2 Discovery Engine — Plan

## Mục tiêu
Biến music server thành trải nghiệm Spotify/Plexamp-like với recommendations và radio hoàn toàn local-first, không dùng thư viện bên ngoài.

---

## Dữ liệu sẵn có để khai thác

| Nguồn | Dùng cho |
|---|---|
| `tracks.play_count` | Weighting, Most Played |
| `tracks.last_played_at` | Recently Played seed |
| `tracks.artist / album` | Artist/Album grouping |
| `favorites.track_id` | User preference signal |
| `play_history.played_at` | "Because You Played" seed |
| `artist_metadata_cache.tags_json` | Genre/tag overlap → Similar Artists |
| `artist_metadata_cache.country / area` | Country match bonus |
| `artist_metadata_cache.listeners / playcount` | Popularity signal |
| `artist_metadata_cache.popular_tracks_json` | Track names (not local IDs) |

## Heuristic Similar Artists

Không có listening graph → dùng metadata overlap:

**Signal 1 — Tag/genre overlap (40%):**
- So sánh `tags_json` giữa target artist và các artist khác trong `artist_metadata_cache`
- Score = số tag chung / tổng tag unique (Jaccard index)
- Chỉ so sánh artists đã có cache (không fetch mới)

**Signal 2 — Có trong library (+30%):**
- Bonus nếu artist có tracks trong `tracks` table → có thể phát nhạc ngay
- Không có tracks = chỉ show info, không play được

**Signal 3 — User đã nghe (+20%):**
- Bonus nếu `SUM(play_count) > 0` cho artist đó trong library
- Nghĩa là user đã tự explore rồi → confident recommendation

**Signal 4 — Cùng country (+10%):**
- So sánh `country` field trong cache

**Fallback — Nếu target artist chưa có cache:**
- Dùng `inferLocalTags()` từ artists.controller để lấy local tags
- Giảm độ chính xác nhưng vẫn có kết quả

## Heuristic Artist Radio

```
1. Lấy tất cả tracks của target artist
2. Lấy top 3 similar artists (có tracks trong library)
3. Thêm tracks của similar artists (tỷ lệ thấp hơn)
4. Weighted shuffle: play_count cao → xác suất xuất hiện cao hơn
5. Trả về tối đa limit tracks, deduplicated
```

---

## Heuristic Album Radio

```
1. Lấy tất cả tracks của album đó
2. Lấy artist của album → similar artists
3. Thêm tracks từ similar artists (20% ratio)
4. Shuffle, return
```

---

## Heuristic Track Radio

```
1. Seed track: artist A, album B, tags T
2. Priority 1: tracks cùng album (trừ seed)
3. Priority 2: tracks cùng artist (trừ album)
4. Priority 3: tracks từ similar artists (tag overlap)
5. Weighted shuffle, return 50 tracks
```

---

## Heuristic Auto Mix

```
Nguồn: 
  - Top 20 most played (play_count DESC)
  - Top 20 favorites (ordered by created_at DESC)
  - 10 recently played (last_played_at DESC)
Merge → deduplicate → shuffle → 50 tracks
```

---

## Heuristic Daily Mix

```
Seed: last 7 ngày nghe nhiều nhất (GROUP BY artist, COUNT)
→ Lấy top 3 artists
→ Với mỗi artist: 10 tracks (mix played + unplayed)
→ Shuffle tổng thể → 30 tracks
```

---

## Heuristic Because You Played

```
Seed: track gần nhất trong play_history
→ Artist radio của artist đó
→ Album radio nếu cùng album
→ Label: "Because you played [Track Title]"
```
